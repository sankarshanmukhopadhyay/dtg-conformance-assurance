# Contributing

## What to contribute
- New risks or refinements to risk statements and scoring
- Control objectives and mappings to NIST / ISO / ToIP layers
- Evidence and test-suite definitions
- Spec edits (DCAS + annexes)

## How
1. File an Issue using the templates in `.github/ISSUE_TEMPLATE/`
2. Create a branch (e.g. `draft/risk-G12-revocation-latency`)
3. Submit a PR; include updated CSV exports for reviewable diffs
